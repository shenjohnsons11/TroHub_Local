Phân Tích Thiết Kế & Chức Năng - TroHub App

Tài liệu phân tích kiến trúc, luồng xử lý và so sánh kỹ thuật ứng dụng quản lý nhà trọ

1. Tổng Quan Kiến Trúc (Architecture Overview)

TroHub là một ứng dụng di động được xây dựng bằng React Native (Expo), sử dụng kiến trúc phân lớp rõ ràng để tách biệt UI và logic gọi API.

📂 Tầng Giao Diện (Screens & Components)

Chứa các file .tsx hiển thị UI. Được chia làm hai luồng chính:

  • Admin: Quản lý phòng, hợp đồng, hóa đơn, người thuê.
  • Tenant (Người thuê): Xem hóa đơn, báo cáo sửa chữa, xem hợp đồng.

⚙️ Tầng Dịch Vụ (Services)

Các file trong thư mục services/ chịu trách nhiệm gọi API, đóng gói dữ liệu và xử lý lỗi trước khi trả về cho UI. Nổi bật là apiClient.ts làm core fetcher.

📦 Tầng Dữ Liệu (Types & Constants)

Sử dụng TypeScript Interfaces để định nghĩa cấu trúc dữ liệu chặt chẽ (vd: Invoice, Contract) và hằng số lưu trữ endpoint, cấu hình màu sắc.

2. Phân Tích Chức Năng Trọng Tâm & Chú Thích Code

A. Core API Client (Xử lý tập trung các request)

Thay vì sử dụng fetch ở khắp mọi màn hình, ứng dụng tạo ra một apiClient.ts để quản lý chung Headers, Token và xử lý lỗi.

import { API_BASE_URL } from "../constants/api";

export const apiClient = {
  async request<T>(endpoint: string, options: ApiOptions = {}): Promise<T> {
    const { method = "GET", body, token } = options;
    
    // 1. Tự động ghép nối API_BASE_URL (localhost) với endpoint cụ thể
    const url = `${API_BASE_URL}${endpoint}`; 

    const response = await fetch(url, {
      method,
      headers: {
        "Content-Type": "application/json",
        // 2. Tự động đính kèm Token nếu yêu cầu xác thực
        ...(token ? { Authorization: `Bearer ${token}` } : {}),
      },
      body: body ? JSON.stringify(body) : undefined,
    });

    const text = await response.text();
    const data = text ? JSON.parse(text) : null;

    // 3. Xử lý lỗi HTTP status tập trung
    if (!response.ok) {
      throw new Error(data?.message || "Có lỗi xảy ra khi gọi API");
    }
    return data as T;
  }
};

B. Quản lý Môi trường API (Localhost vs Server)

Code đã được chỉnh sửa để tự động nhận diện thiết bị và trỏ về Localhost thay vì Server cứng.

import { Platform } from "react-native";

// 👉 CHÚ THÍCH QUAN TRỌNG: 
// Android Emulator (Máy ảo) dùng 10.0.2.2 để gọi về localhost của máy tính.
// iOS/Web dùng 'localhost'. Thiết kế này giúp không cần sửa code khi đổi thiết bị test.
const LOCAL_IP = Platform.OS === 'android' ? "10.0.2.2" : "localhost";

export const API_BASE_URL = `http://${LOCAL_IP}:3000/api`;

3. So Sánh Luồng Xử Lý: API Server vs API Localhost

Mặc dù giao diện và chức năng 100% giống nhau, nhưng môi trường chạy API (Server Thực tế so với Localhost) mang lại đặc thù luồng xử lý khác biệt trong lúc phát triển và kiểm thử.

Tiêu chí Chạy API trên Server (Bản cũ) Chạy API Localhost (Bản mới)
Độ trễ (Latency) Có độ trễ do mạng Internet (50ms - 200ms+). Flow UI cần handle Loading state kỹ lưỡng. Gần như 0ms. Trả về kết quả tức thì. Phù hợp để code logic và UI nhanh chóng.
Phụ thuộc Mạng Cần có Internet. Server sập hoặc mạng yếu sẽ không test được App. Offline 100%. Không cần WiFi vẫn code và test mọi luồng CRUD bình thường.
Backend Code Khó sửa trực tiếp nếu phát hiện lỗi API (phải deploy lên server lại). Sửa code backend (NodeJS) ở máy và test ngay trên App lập tức không qua bước deploy.
Bảo mật & CORS Phải cấu hình CORS, HTTPS nghiêm ngặt. Localhost thường bỏ qua các lỗi CORS/SSL gắt gao, dễ dàng debug.

4. Ví Dụ Luồng Xử Lý Logic Cốt Lõi (Login & Fetch Data)

Luồng Đăng Nhập (Login Flow)

LoginScreen
(Nhập User/Pass)
➔
authService.ts
(Gọi API POST)
➔
Local API (Port 3000)
(Validate DB)
➔
Return Token
(Lưu SecureStore)
➔
Điều hướng
(HomeScreen / AdminScreen)

Sự đồng bộ trong Luồng:

  • Giống nhau: Logic điều hướng, validate mật khẩu rỗng ở Front-end, catch try/catch khi gọi API đều y hệt nhau. Dù là local hay server, luồng UI state (isLoading, isError) không thay đổi.
  • Khác biệt ngầm: Khi chạy local, bước Local API (Port 3000) xử lý kết nối tới DB Local (như MySQL/MongoDB chạy trên XAMPP/Docker ở máy bạn), thay vì kết nối tới Database trên Cloud AWS/DigitalOcean. Do đó bạn có thể reset dữ liệu test cực kỳ dễ dàng.
