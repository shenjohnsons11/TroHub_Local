🚀 Hướng Dẫn Deploy & Test Chung Qua Docker

Tình Trạng Hiện Tại

Repo này (TroHub_Local) hiện tại chỉ chứa Frontend (Giao diện App). Khi bạn chạy Docker ở đây, nó sẽ mở ra giao diện Web Admin.

⚠️ Lưu ý quan trọng: Để đồng nghiệp của bạn có thể đăng nhập và lấy "data local", thao tác đóng Docker phải bao gồm cả Backend (API) và Database (MySQL/MongoDB...). Nếu họ chỉ chạy cái Frontend này, gọi API sẽ báo lỗi mạng do máy họ không có cục Database của bạn!

Phương Án 1: Chia Sẻ Nhanh Không Cần Docker (Qua Mạng LAN)

Nếu bạn và đồng nghiệp ngồi chung một WiFi, bạn không cần phải gửi file gì cho họ cả. Làm như sau:

  1. Bật Backend và Database trên máy tính của bạn.
  2. Bật Frontend này (npm run web).
  3. Đổi địa chỉ API trong file constants/api.ts từ localhost thành địa chỉ IP máy bạn (ví dụ: 192.168.1.12).
  4. Gửi đường link http://192.168.1.12:8081 cho đồng nghiệp. Họ mở trình duyệt lên là dùng luôn data trên máy bạn!

Phương Án 2: Đóng Gói Docker Gửi Cho Đồng Nghiệp (Chạy Máy Riêng)

Nếu đồng nghiệp ở xa, bạn phải đóng gói cả hệ thống. Tôi đã tạo sẵn file docker-compose.yml cho bạn.

Bước 1: Chuẩn bị Source Code Backend

Hãy copy thư mục Backend của bạn bỏ chung vào với thư mục chứa Frontend này, hoặc copy file docker-compose.yml ra ngoài một thư mục cha chứa cả hai.

Bước 2: Cấu Hình File docker-compose.yml

Mở file docker-compose.yml lên. Bạn sẽ thấy tôi đã comment (ẩn đi) phần cấu hình Backend và Database. Hãy bỏ comment và sửa lại thông số cho đúng với mã nguồn backend của bạn.

Bước 3: Tạo Data Mẫu (Seed Data)

Để máy bạn và đồng nghiệp có chung data, bạn hãy export Database trên máy bạn ra một file init.sql (nếu dùng MySQL/Postgres), hoặc code sẵn một logic tự động tạo User/Admin khi Backend khởi động lần đầu.

Bước 4: Chạy Lệnh Khởi Động

Gửi toàn bộ thư mục (Frontend + Backend + file docker-compose) cho đồng nghiệp. Họ chỉ cần cài đặt phần mềm Docker Desktop và chạy lệnh sau trong Terminal:

docker-compose up --build -d

Chờ vài phút, sau đó truy cập http://localhost:8081 là giao diện và mọi thứ sẽ hoạt động chung một database độc lập trên máy họ!
